Quick Start

De cero a tu portfolio corriendo en menos de 5 minutos.

Requisitos

  • Node.js 22+ — verifica con node -v
  • pnpm 9+ — instala con npm i -g pnpm
  • VS Code — con la extensión oficial de Astro

Instalación

Prerequisites: This template uses pnpm instead of npm. Install it first: npm install -g pnpm (or other install methods)
Terminal 
# 1. Install dependencies
pnpm install

# 2. Start dev server
pnpm dev

# 3. Open http://localhost:4321 in your browser
¿Error en el dev server? Ejecuta rm -rf .astro node_modules/.vite && pnpm dev para limpiar cachés de Vite y Astro.

Primeros pasos

  1. Edita src/site-config.ts — pon tu nombre, email y redes sociales
  2. Reemplaza public/favicon.svg con tu logo
  3. Reemplaza public/og-image.jpg con tu imagen social (1200×630px)
  4. Edita el contenido en src/content/ — agrega tus proyectos reales

site-config.ts

El único archivo que necesitas editar para personalizar el 80% del sitio. Cada campo documentado abajo.

site — Core

Campo Tipo Qué hace Ejemplo
url string Tu dominio. Sin barra final. Crítico para SEO y sitemaps. "https://miportfolio.dev"
lang string Atributo lang del HTML. "es", "en"
locale string Formato de fechas. "es-CL"
name string Nombre comercial. Logo + copyright. "dev.designer"
favicon string Ruta del favicon en /public. "/favicon.svg"

profile — Tu identidad

Campo Tipo Qué hace Ejemplo
name string Tu nombre completo. SEO author + JSON-LD. "Juan Alfaro"
role string Tu cargo. JSON-LD jobTitle. "Frontend Developer"
headline string Propuesta de valor corta. Se muestra en el Footer. "Construyendo el futuro..."
bio string Tu biografía. Se renderiza como párrafo del Hero. "Especializado en crear..."
avatar string Tu foto. Se guarda en /public. Si no existe, muestra iniciales. "/images/avatar.webp"
heroImage string Foto principal del Hero. Va en /src/assets/ para compresión automática. "/src/assets/images/..."

seo — Metadatos

Campo Tipo Qué hace
titleSuffix string Se añade a cada título de página.
defaultDescription string Meta description fallback.
ogImage string Imagen para Open Graph / Twitter Cards. 1200×630px.
twitterHandle string Tu usuario de X/Twitter.
themeColor object Colores para mobile browsers (dark/light).

contact — Contacto

Campo Tipo Qué hace
email string Tu email. Se usa en la página de contacto, formulario y legales.
calCom string Enlace a Cal.com para agendar llamadas.
formEndpoint string Backend del formulario. Pon la URL de Formspree o Cloudflare Forms. Si está vacío, el formulario muestra una pista de configuración.
URL sin barra final. El campo site.url no debe terminar en /. "https://miportfolio.dev" ✅ "https://miportfolio.dev/" ❌

Colores & Temas

El portfolio usa colores HSL en formato string. El color primario se aplica a botones, links y acentos.

Cómo cambiar tu color de marca

  1. Inicia pnpm dev
  2. Haz clic en el ícono de paleta flotante (esquina inferior derecha)
  3. Elige una paleta o usa el selector de color
  4. Haz clic en "Copy" en el panel
  5. Pega el objeto en site-config.ts → ui.brandColor
site-config.ts — ui.brandColor 
brandColor: {
  light: { primary: "152 70% 38%", foreground: "0 0% 100%" },
  dark: { primary: "152 70% 58%", foreground: "0 0% 100%" }
}

Variables CSS disponibles

En src/styles/global.css se definen todos los tokens: --background, --foreground, --primary, etc. Cada uno tiene valores distintos para :root (light) y .dark.

i18n (Idiomas)

Sistema de internacionalización integrado. El idioma se controla desde site.lang en site-config.ts.

Idiomas soportados

Código Idioma Locale (fechas)
es Spanish (Español) es-CL, es-ES, es-MX...
en English en-US, en-GB...
pl Polish (Polski) pl-PL

Cómo cambiar el idioma

site-config.ts 
site: {
  lang: 'en',      // 'es' | 'en' | 'pl'
  locale: 'en-US', 
}
Nota: El sitio usa un solo idioma global. Para activar el idioma polaco, cambia ambos valores.
Navegación internacionalizada: Los ítems de navigation y legal en site-config.ts usan claves de traducción (nav.home, nav.projects, legal.privacy…). Header.astro y Footer.astro los resuelven con t() automáticamente al cambiar site.lang.

Agregar un nuevo idioma

  1. Abre src/i18n.ts
  2. Agrega un nuevo objeto en translations (ej: fr: { ... } para Francés)
  3. Copia todas las claves de en y traduce los valores
  4. Actualiza SupportedLocale para incluir el nuevo código
  5. Usa site.lang: 'fr' en site-config.ts

Header & Logo

Personaliza el logo en el header. Puedes usar texto (por defecto), un logo SVG, o una imagen.

site-config.ts — header 
header: {
  // 'text' | 'svg' | 'image'
  logoType: 'text',
  logoSvg: '',
  logoImage: '',
  logoAlt: 'Logo',
},

Opciones de header.logoType

Propiedad Tipo Descripción
logoType 'text' | 'svg' | 'image' Tipo de logo a mostrar en el header.
logoSvg string Ruta al archivo SVG en public/ (ej: '/logo.svg'). Solo para logoType: 'svg'.
logoImage string Ruta a la imagen (ej: '/images/logo.png'). Solo para logoType: 'image'.
logoAlt string Texto alternativo para accesibilidad (atributo alt).

Ejemplos

Logo SVG:

site-config.ts 
header: {
  logoType: 'svg',
  logoSvg: '/logo.svg',
  logoAlt: 'Mi Empresa',
},

Logo imagen (PNG/WebP):

site-config.ts 
header: {
  logoType: 'image',
  logoImage: '/images/logo.webp',
  logoAlt: 'Mi Empresa',
},

Texto (por defecto):

site-config.ts 
header: {
  logoType: 'text', // Usa site.name para generar el logo
},
Tip: Coloca tus archivos de logo en la carpeta public/ para que sean servidos estáticamente. SVG es preferible por ser escalable y ligero.

Footer & Logo

Personaliza el logo en el footer. Funciona igual que el header: puedes usar texto (por defecto), un logo SVG, o una imagen. Además incluye el campo text para el texto de versión/copyright.

site-config.ts — footer 
footer: {
  // 'text' | 'svg' | 'image'
  logoType: 'text',
  logoSvg: '',
  logoImage: '',
  logoAlt: 'Logo',
  text: 'v2.9.0',
},

Opciones de footer

Propiedad Tipo Descripción
logoType 'text' | 'svg' | 'image' Tipo de logo a mostrar en el footer.
logoSvg string Ruta al archivo SVG en public/. Solo para logoType: 'svg'.
logoImage string Ruta a la imagen. Solo para logoType: 'image'.
logoAlt string Texto alternativo para accesibilidad.
text string Texto de versión/copyright que aparece junto al nombre del sitio en la barra inferior del footer.

Ejemplos

Logo SVG en footer:

site-config.ts 
footer: {
  logoType: 'svg',
  logoSvg: '/logo.svg',
  logoAlt: 'Mi Empresa',
  text: 'v2.9.0',
},

Logo imagen en footer:

site-config.ts 
footer: {
  logoType: 'image',
  logoImage: '/images/logo.webp',
  logoAlt: 'Mi Empresa',
  text: 'v2.9.0',
},

Texto (por defecto):

site-config.ts 
footer: {
  logoType: 'text', // Usa site.name para generar el logo
  text: 'v2.9.0',
},
Tip: Puedes usar el mismo archivo SVG tanto en el header como en el footer. Si tu marca tiene versiones distintas (horizontal/compacta), usa rutas diferentes en cada sección.

Toggles de Interfaz

Activa o desactiva features desde site-config.ts → ui. Ponlo en false para desactivarlo.

Toggle Qué controla Default
enableViewTransitions Transiciones SPA entre páginas (sin recarga). true
showThemePicker Panel flotante de colores. Desactívalo en producción. true
showBuyBanner Banner "Comprar Plantilla". Desactívalo para el cliente final. true
enableCustomCursor Cursor interactivo premium con efecto magnético. true
showBackgroundPattern Patrón de puntos (dot grid) en el fondo. false para fondo limpio. true
showBackgroundOrbs Orbes de color/aurora en el fondo. Combina con showBackgroundPattern: false para fondo blanco. true

Control de Tema (Dark/Light)

La sección theme en site-config.ts te da control total sobre el modo oscuro/claro. Puedes permitir que el usuario elija o forzar un tema específico.

site-config.ts — theme 
theme: {
  // Si es false, el usuario NO puede cambiar el tema
  allowToggle: true,
  // 'light' o 'dark' — tema por defecto
  defaultMode: 'dark',
},
Propiedad Tipo Descripción
allowToggle boolean Si es false, el botón de tema desaparece y se bloquea en defaultMode.
defaultMode 'light' | 'dark' Tema forzado cuando allowToggle: false. Ignora preferencias del sistema.

Ejemplos de uso

Tema forzado a light (sin toggle):

site-config.ts 
theme: {
  allowToggle: false,
  defaultMode: 'light',
},

Toggle habilitado (comportamiento por defecto):

site-config.ts 
theme: {
  allowToggle: true,
  defaultMode: 'dark', // Solo afecta carga inicial
},
Migración desde enableDarkMode: La opción ui.enableDarkMode fue reemplazada por la sección theme. Si tenías enableDarkMode: false, usa allowToggle: false, defaultMode: 'light' en su lugar.

Colecciones de Contenido

Tu contenido vive en src/content/ como archivos MDX con frontmatter validado por Zod. Si omites un campo obligatorio, Astro fallará en el build.

Colección Ruta Formato Para qué
Proyectos src/content/projects/ MDX Casos de estudio del portfolio
Blog src/content/blog/ MDX Artículos y reflexiones
Servicios src/content/services/ MDX Páginas de servicios / ofertas
Testimonios src/content/testimonials/ MD Reseñas de clientes
FAQ src/content/faq/ MD Preguntas frecuentes del accordion
Changelog src/content/changelog/ MD Historial de versiones
¿Cómo se nombra un archivo? Usa kebab-case: mi-proyecto.md → la URL será /projects/mi-proyecto.

Proyectos

src/content/projects/mi-proyecto.md

---

Aquí va el cuerpo del proyecto en Markdown o MDX.
Puedes incluir imágenes, código y lo que necesites.
Campo Requerido Tipo Notas
title Sí string —
description Sí string Resumen corto para la tarjeta.
technologies Sí array Debe ser ["A", "B"], no un string.
date Sí date Formato YYYY-MM-DD. Ordena del más nuevo al más viejo.
repo No URL Si existe, muestra ícono GitHub.
live No URL Si existe, muestra enlace "Ver en vivo".
image No ruta Apunta a /src/assets/ para compresión automática.

Imágenes

Dos ubicaciones con propósitos distintos.

/public/images/

Assets que se sirven tal cual. Avatares, logos. No se optimizan.

/src/assets/images/

Imágenes que Astro procesa y optimiza automáticamente (webp, responsive, tamaños). Úsalas para hero y proyectos.

Formato recomendado: WebP para fotos, SVG para logos e íconos. Tamaño máximo de OG image: 1200×630px.

Tipografía

Se usan dos fuentes locales (sin Google Fonts, sin tracking):

  • Inter — Sans-serif principal. Importada desde @fontsource-variable/inter.
  • Roboto Mono — Monospace para código y etiquetas. Importada desde @fontsource-variable/roboto-mono.

Ambas se importan en src/layouts/BaseLayout.astro y se configuran en tailwind.config.mjs.

Formulario de Contacto

El formulario envía datos reales. Solo necesitas configurar un endpoint.

Opción 1: Formspree (Recomendado)

  1. Crea cuenta en formspree.io
  2. Crea un formulario nuevo y copia la URL
  3. Pégala en site-config.ts → contact.formEndpoint

Opción 2: Cloudflare Forms / Custom API

Cualquier endpoint que acepte POST con FormData. Configura contact.formEndpoint con tu URL.

site-config.ts 
contact: {
  email: "hola@miportfolio.dev",
  formEndpoint: "https://formspree.io/f/xzyabc",  // ← tu URL aquí
  calCom: "https://cal.com/tuusuario"
}
Anti-spam incluido: El formulario incluye un campo honeypot invisible (_gotcha) que bloquea bots.

Paginación

Configura cuántos items mostrar antes de crear nuevas páginas.

site-config.ts 
content: {
  projectsPerPage: 6,
  postsPerPage: 4
}

Esto genera automáticamente:

  • /projects — página 1 (6 proyectos)
  • /projects/page/2 — página 2 (si hay más de 6)
  • /blog — página 1 (4 posts)
  • /blog/page/2 — página 2 (si hay más de 4)

SEO & GEO

El componente SEO.astro genera todo automáticamente. Tú solo llenas el config.

Qué genera automáticamente

  • Meta tags: title, description, keywords, author, canonical
  • Open Graph: og:type, og:title, og:description, og:image (1200×630), og:locale
  • Twitter Cards: summary_large_image con todos los campos
  • JSON-LD: Person (home), Article (blog), CreativeWork (projects), FAQPage (faq), Service (services)
  • Sitemap: sitemap-index.xml generado en cada build
  • Robots.txt: Ruta dinámica que referencia el sitemap

GEO (Generative Engine Optimization)

El archivo public/llms.txt describe tu sitio para motores de búsqueda con IA (Perplexity, ChatGPT Search, Claude). Contiene: stack, estructura de carpetas, colecciones de contenido y enlaces rápidos.

Accesibilidad

El template cumple con WCAG 2.2 AA. Estas son las protecciones activas:

Reduced Motion

Si el usuario tiene prefers-reduced-motion: reduce, todas las animaciones (reveal, hover, magnetic, mobile menu) se desactivan automáticamente.

Navegación por teclado

El mobile menu se cierra con Escape. Todos los botones tienen aria-label y los iconos sociales tienen sr-only text.

Contraste AA

Los tokens HSL están calibrados para cumplir WCAG AA en ambos modos (light y dark).

Alt texts descriptivos

Las imágenes del Hero incluyen nombre, rol y contexto. Los proyectos usan el título como alt fallback.

PWA

Progressive Web App con service worker auto-update.

Qué incluye

  • site.webmanifest en /public/
  • Service worker con estrategia de caché para .js, .css, .html, .ico, .png, .svg, .webp, .avif
  • Registro automático con autoUpdate

Analytics

El template viene sin tracking por defecto. Estos son los proveedores soportados:

Proveedor Tipo Setup
Cloudflare Web Analytics Zero-JS, server-side Actívalo en Cloudflare Dashboard. Sin código.

Deploy en Cloudflare Pages

La opción recomendada. Astro + Cloudflare = edge deployment global.

Desde GitHub (recomendado)

Paso 1: Sube tu código

Haz commit y push a tu repositorio: git push origin main

Paso 2: Conecta en Cloudflare

Ve a dash.cloudflare.com → Workers & Pages → Create → Connect to Git → selecciona tu repo.

Paso 3: Configura el build

Usa el preset Astro o configúralo manualmente:

  • Build command: pnpm build
  • Build output directory: dist
  • Node version: 22
Node 22 es obligatorio. En Cloudflare Pages, agrega la variable de entorno NODE_VERSION = 22 en Settings > Environment Variables.

Dominio personalizado

En Cloudflare Pages → tu proyecto → Custom domains → agrega tu dominio. Cloudflare maneja SSL automáticamente.

Variables de Entorno

Copia .env.example a .env y configura según necesites.

.env 
# Contact form backend
PUBLIC_CONTACT_FORM_ENDPOINT=https://formspree.io/f/xxxx

# Analytics (elige uno)
PUBLIC_PLAUSIBLE_DOMAIN=miportfolio.dev
PUBLIC_GA_MEASUREMENT_ID=G-XXXXXXXXXX
Nunca comitees .env. El archivo está en .gitignore. Usa .env.example como referencia para el equipo.

Troubleshooting

Errores comunes y sus soluciones.

"Dev server crashes with Cloudflare worker error"

Caché corrupto de Vite/Astro. Ejecuta:

rm -rf .astro node_modules/.vite && pnpm dev

"CommonJS dependency breaks the build"

Agrega la dependencia a ssr.noExternal en astro.config.mjs: 

vite: {
  ssr: { noExternal: ['your-package-name'] }
}

"Zod validation error en content"

Revisa el frontmatter del archivo MDX que el error menciona. Un campo obligatorio está faltando o tiene el tipo incorrecto.

"El formulario de contacto no envía"

Verifica que contact.formEndpoint en site-config.ts tenga una URL válida. Si está vacío, el formulario muestra un aviso de configuración.

"PWA no funciona en producción"

Asegúrate de que el manifest en astro.config.mjs tenga la URL correcta. Ejecuta node scripts/generate-pwa-icons.mjs para generar los PNGs necesarios.

"Cambios en site-config.ts no se reflejan"

El dev server de Astro a veces cachea el config. Reinicia el servidor: Ctrl+C → pnpm dev.
